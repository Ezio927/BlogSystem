# V1 需求基线

## 1. 产品定义

Open Blog 是一套本地优先的开源个人博客系统。博客拥有者通过仅在本机运行的 Studio 创建、编辑、预览和发布 Markdown 内容；访客通过部署在用户自有服务器上的 Web 应用访问博客。

V1 面向具备基础开发和部署能力的用户，不承诺零代码一键建站。

## 2. 核心流程

```text
本地 Studio 编辑 → 本地预览 → 内容校验 → Git 提交/推送
                                           ↓
                                    CI 构建与测试
                                           ↓
                                    Docker 更新公网博客
```

Studio 关闭后不得影响公网博客运行。

## 3. Must Have

### 公网博客

- 首页、文章列表、文章详情；
- 标签、归档和关于页面；
- Markdown/MDX 内容、代码高亮、数学公式和文章目录；
- 响应式布局与深色模式；
- 基础 SEO、RSS、sitemap、robots.txt 和 404；
- 上一篇/下一篇导航。

### 本地 Studio

- 新建、编辑和删除文章；
- 草稿与已发布状态；
- Markdown 编辑与实时预览；
- 标题、slug、摘要、标签、封面、发布时间等元数据表单；
- 本地图片导入；
- 博客名称、头像、简介和社交链接配置；
- 发布前校验、构建检查与可读错误信息。

### 工程与部署

- 单一内容源，不在 Web 和 Studio 间复制文章；
- Git 版本管理与 Pull Request 协作；
- lint、类型检查、测试和构建 CI；
- Docker 部署与反向代理示例；
- 环境变量、备份和恢复文档；
- Studio 默认只监听本机地址。

## 4. Should Have

- 全文搜索；
- 阅读时间和文章封面；
- 友链与项目展示；
- 简单的第三方评论接入；
- 发布日志和 Docker 健康检查；
- 配置导入与导出。

## 5. Could Have

- 音乐播放器；
- 天气组件；
- 弹幕与粒子特效；
- 图床服务；
- 复杂评论系统；
- 访问量统计后台；
- 照片墙与动态；
- AI 助手；
- 多主题；
- Vercel 一键部署。

这些能力通过扩展契约和页面插槽接入，不得耦合进内容核心。

## 6. Won't Have in V1

- 官方托管服务和多租户；
- 公网管理后台与用户注册系统；
- 插件市场与主题市场；
- 移动端 App；
- 多人实时协同编辑；
- 自动购买服务器或域名；
- 面向零基础用户的桌面安装包。

## 7. 非功能要求

- `main` 始终可构建、可部署；
- 不允许通过忽略 TypeScript 错误完成构建；
- 真实密钥不得进入 Git 或浏览器端代码；
- 用户内容应能独立备份和迁移；
- 核心内容格式的破坏性修改必须有迁移说明；
- Studio 的文件和 Git 操作需要限定在用户选择的博客目录内。

## 8. V1 验收标准

一位未参与开发、但懂基础 Git 和 Docker 的测试者，能够根据文档完成：

1. 安装依赖并启动 Web 与 Studio；
2. 修改站点资料；
3. 创建一篇带图片的文章并预览；
4. 将文章发布到 Git 仓库；
5. 通过 CI 更新 Docker 博客；
6. 更新文章并再次发布；
7. 备份内容并在干净目录中恢复。
